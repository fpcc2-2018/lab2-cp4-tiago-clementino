Abstract

A exploratory report involving an analysis task proposed by Wikimedias Foundation. The work described here aims to find specific relationships between the properties present in the data using confidence intervals and bootstrap. The result is some useful conclusions that may be reached only on the basis of the available information in the sample.

Objetivo

Vamos reproduzir um problema de explorao de dados que a Fundao Wikimedia usou em 2016 no recrutamento de pessoas para trabalhar em seu departamento de descoberta. O objetivo seguir as instrues colocadas na tarefa original, respondendo as 4 perguntas que so colocadas.

Neste laboratrio, o objetivo explorar as possibilidades da inferncia estatstica por intervalos de confiana. A entrega consiste de um relatrio explorando por inferncia em IC as trs primeiras perguntas do problema proposto pela Wikimedia. Alm de uma anlise da pergunta 01, agora comparando o grupo A internamento (dividido de forma aleatria), tambm por inferncia em IC.

Contexto

O departamento de descoberta da Fundao Wikimedia utiliza um registro de eventos (Event Logging, EL) para avaliar vrias mtricas de performance e usabilidade, ajudando-os a tomar decises. Principalmente, este departamento est interessado em:

  • clickthrough rate: a proporo de sesses de busca onde o usurio clica em um dos resultados apresentados.

  • zero results rate: a proporo de buscas que no tiveram resultados.

e outras mtricas fora do escopo desta tarefa. EL usa Javascript para enviar mensagens aos servidores da Wikimedia de forma assncrona quando o usurio realiza aes especficas. Nesta tarefa irei analisar um subconjunto de nosso registro de eventos.

Dados

O conjunto de dados vem de uma estratgia de EL usada para avaliar a satisfao do usurio. Usurios standalone so randomicamente coletados para ser anonimamente rastreados por tal estratgia, que usa um sistema de descoberta do tipo Im alive para estimar quanto tempo os usurios permanecem nas pginas que eles visitam. O conjunto de dados contm em tabela pouco mais de uma semana de EL com as seguintes colunas.

Campo Tipo Descrio

Questes

Q1. Qual a clickthrough rate geral diria? Como isto varia entre os grupos?

Algo crucial para comearmos a responder esta questo calcular a taxa de cliques por sesso de busca. Como a taxa por sesso, agrupar os dados desta forma um bom primeiro passo. Enquanto ocorre este agrupamento criamos uma nova varivel que ajudam na tarefa de analisar a taxa de cliques: visited para identificar facilmente qual sesso teve ou no cliques (1 ou 0). results e num_clicks somam as variveis originais de mesmo nome, agrupando por sesso.

sessions = buscas %>% #agrupando por sesso
    group_by(group, day, session_id) %>% 
    summarise(visited = ifelse(sum(num_clicks) > 0, 1, 0),
              session_legth = first(session_legth),
              last_search = first(last_search),
              results = sum(results, na.rm = TRUE),
              num_clicks = sum(num_clicks, na.rm = TRUE), 
              first_search_zero = ifelse(first(results)==0, "Sim", "No"))

Aqui, partindo da premissa que os dados em questo dizem respeito a uma amostra e utilizando o mtodo bootstrap, podemos estimar a distribuio amostral da mdia da varivel visited geral e por grupo. Alm disto, para observar uma diferena pareada entre os grupos a e b, vamos estimar tambm a diferena entre as mdias de a e b para visited, observe:

funcao_bootstrap_1_geral <- function(dado, indices){
    d = dado %>% 
      slice(indices) %>% 
      summarise(click_through = mean(visited)) %>% 
      pull(click_through)   
    return(mean(d[1]))
}
